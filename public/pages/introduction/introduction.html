Node.js & NPM

Node.js is an asynchronous event-driven JavaScript runtime environment for building scalable web applications. Its asynchronous and event-driven nature enables it to handle multiple requests simultaneously, eliminating the need to wait for one task to finish before starting another. With Node.js, developers can seamlessly create both front-end and back-end components using a single language, eliminating the need to learn additional languages for full-stack development.

npm (Node Package Manager) serves as the primary package manager for Node.js, offering a vast repository of open-source software. Utilizing npm, developers can easily download and install pre-built packages into their projects, streamlining the development process and ensuring code stability through tested packages.

Installation

To get started with Node.js, simply install it from Node.js's website. Npm is automatically installed alongside Node.js. You can check the installed versions using node -v and npm -v in your terminal. 


        PS C:\Users\jespe\Desktop\4.Semster\Node\NodeMandatoryI> node -v
        v20.11.0

        PS C:\Users\jespe\Desktop\4.Semster\Node\NodeMandatoryI> npm -v
        10.2.4

npm offers two installation methods: local and global. Local installation ensures that packages are confined to the project folder, aiding deployment and preventing conflicts. To install a package locally, use npm install , for example, npm install express.

Global installation makes packages accessible across all project folders. To install a package globally, use npm install -g , for example, npm install -g nodemon.

Variables

In simple terms, a variable is container that holds a value and points to the storage location where the data is stored. This value can me changed and modified throughout the execution of the program. In JavaScript we can declare variable by using the keywords “var”, “let, “const”, followed by the variable name. But due the difference in scooping there was a shift from using “var” to “let and “const”.

Before the release of ES6 (ECMAScript 2015), var was the primary keyword used for variable declaration. Any variable declared with `var` are function- or globally scooped, which means they are accessible thought the entire function they are declared with no regards for the block within the function. 


        function example() {
          if(true) {
            var x = 10;
          }
          console. log(x);
        }
        example();

        // Output: 10

After the introduction of ES6 `let` and `const` was introduced. Variable declared with `let` and `const` are blocked scooped, meaning they only exist inside the block that is enclosed my curly braces in where they are declared. The difference in `let` and `const` is that `let` allows variable values to be reassigned and `const` does not allow this once the value has been assigned. 


        function example() {
          if(true) {
            let y = 20;
            const z = 30;
          }
          console. log(y);
          console. log(z);
        }
        
        example();

        // Throws ReferenceError: y is not defined

Because of the difference in scooping, `let` and `const` are the preferred over `var`. The block scooping provides better control over variables and reduces the risk of unintended side effect ands bugs. `const` also helps prevent accidental re-assignment of variable, when you know that the value should not be changed after initialization, which is a common source of bugs. 


        function example() {
          let y;
          if(true) {
            y = 20;
          }
          console. log(y);
        }
        
        example();

        // Output: 20
      

        function example() {
          let y;
          if(true) {
            y = 20;
          }
          console. log(y);
        }
        
        example();

        // TypeError: Assignment to constant variable - z = 20;

Datatypes

In JavaScript data types represent the kind of value that a variable can hold and there are several types. See my Gtihub for the exercises revolving this topic.

Primitive Data Types:

  • String: Represents textual data, enclosed within single ('') or double ("") quotes.
  • Number: Represents numeric values, including integers and floating-point numbers.
  • Boolean: Represents true or false values.
  • Undefined: Represents a variable that has been declared but not assigned a value.
  • Null: Represents the intentional absence of any value.
  • Symbol: Represents unique identifiers

        const name = "Jesper"
        const age = 30
        const isStudent = true
        let notDefined;
        const noValue = null

        console.log(typeof name); // string
        console.log(typeof age); // number
        console.log(typeof isStudent); // boolean
        console.log(typeof notDefined); // undefined
        console.log(typeof noValue); // object (bug)
        console.log(typeof noValue === null); // true

Reference Data Types:

  • Object: Represents a collection of key-value pairs, often used for complex data structures.
  • Array: Represents a collection of elements, each identified by an index.
  • Function: Represents a reusable block of code that performs a specific task.

        let person = { firstName: "John", lastName: "Doe" } // object
        let numbers = [1, 2, 3, 4, 5]; // array
        
        function greet() { // function
          console.log("Hello there!");
        }
        
        console.log(typeof person); // Output: object
        console.log(typeof numbers); // Output: object
        console.log(typeof greet()); // Output: function

Type Coercion

Type Coercion is the automatically conversion of values from one data type to another. This process is performed by the JavaScript Engine and occurs when input of different data types is used in an operation.

This function takes in two parameters where both are expected to be strings, but when the function is called “5” as a string and `5` as number is passed. Type coercion will happened and converts the number to a string, which will result in “55” 


        function concatenateStrings(str1, str2) {
          return str1 + str2;
        }
        
        console.log(concatenateStrings("5", 5));
        
        // output: 55

How to avoid

There are different approaches of avoid type coercion. One way is using the Strict Comparison Operators `===` and `!==`. This will compare both the value and the type without performing type coercion. 


        console.log("5" === 5);

        // Output: false

You can also use Explicit Type Convension (type casting) instead. Here you convert the values to the desired data type using functions like `parseInt` or `parseFloat`. 


        console.log(parseInt("5") + 5);

        // Output: 10

API Design

Using the right HTTP verb:

HTTP verbs are used to indicate the action to be perfomed. Choosing an appropiate HTTP-verb for each endpoint ensures that we are following the RESTful principles. The commenly used HTTP-verbs in RESTfull API are:

  • GET: Retrive data from server.
  • POST: Creates a new resource.
  • PUT: Updates an existing resource.
  • PATCH: Partially updates an existing resource.
  • DELETE: Deletes a resource.

Ordering:

In our back-end, the ordering of our endpoint is important because it maintain consistency and structure our enpoints with their associated functionalities. Maintaining this convention helps developers understand and use the API more effectively and reduce confusion.

Naming (noun, plural):

Resource naming convention plays a very important role in RESTful API design, because it represents the entities being manipulated, such as users, drinks, dogs etc. Resource names should be nouns and should typically be plural to represent a collecton. For example if you have a resource representing drink, the endpoint can be "/drinks" to represent a collection of drinks.
